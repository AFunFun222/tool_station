# AI 驱动"千人千面"游戏综合工具站 — 项目规划文档

> 首发打样：《幻兽帕鲁 1.0 正式版》
> 团队规模：5人（2产品 + 2技术 + 1运营）
> 赛制：2天1夜 Hackathon

---

## 一、项目概述

### 1.1 一句话定位

打造一个 AI 驱动的游戏工具站，通过自动化爬取、清洗全网优质内容，根据玩家身份标签（新手/老手）和即时意图，个性化分发结构化攻略与核心数据工具，打破传统 Wiki 千篇一律的呆板展现形式。

### 1.2 为什么是幻兽帕鲁

| 维度 | 数据 |
|------|------|
| 上线时间 | 2026年7月10日，1.0正式版全球同步发售 |
| 玩家规模 | 全平台超 3200 万玩家（截至2025年2月） |
| Steam销量 | 突破 1200 万套，好评率 94% |
| 峰值在线 | 210 万（Steam历史TOP级） |
| 1.0更新规模 | 27页更新日志（常规版本5-9倍），"史上最大规模更新" |
| 新增帕鲁 | 50+ 种（现有近200种，大幅扩充） |
| 新增区域 | 世界树、空岛云海、第二座主岛，地图面积翻倍 |
| 内容生态 | B站帕鲁图鉴合集播放 4.2 亿次，攻略合集 3.8 亿次 |
| 平台覆盖 | PC(Steam)、PS5、Xbox Series X|S、Xbox One、Mac |

核心判断：1.0 正式版是一次现象级内容爆发节点。大量老玩家回坑 + 新玩家涌入，信息需求在7月10日后将井喷。现有工具站全部是静态、千人一面的 Wiki + 计算器，无法应对"信息爆炸 + 人群分层"的双重挑战。

### 1.3 不做会怎样

- 玩家继续在 B站、TapTap、17173、知乎、SMZDM 等平台间反复跳转，效率极低
- 新手被 27 页更新日志和 200+ 帕鲁劝退，老手找不到 1.0 变更对自己存档的影响
- 配种、基地自动化等核心玩法的数据工具散落在多个小站，体验割裂
- 没有任何工具能回答"我现在的进度，下一步该干什么"

## 二、市场机会与竞品分析

### 2.1 现有工具生态全景

| 类型 | 代表 | 核心功能 | 优势 | 痛点 |
|------|------|----------|------|------|
| 静态工具站 | palworldbreedingcalculator.cc | 配种计算器、Tier List、帕鲁数据库、地图 | 功能聚焦、免费无注册 | 数据更新慢、无个性化、1.0新内容空白 |
| 传统Wiki | palworld.fandom.com | 图鉴、掉落、属性数据 | 内容全 | 千人一面、信息过载、无工具属性 |
| 视频攻略 | B站UP主 | 实战演示、配种路线、基地教学 | 直观生动 | 检索困难、无法交互、时效性差 |
| 社区论坛 | TapTap、17173、贴吧 | 讨论、问答、攻略帖 | 社区活跃 | 信息碎片化、质量参差、无结构化 |
| 数据站 | 各类小站 | 掉落表、刷新点、属性计算 | 单点数据 | 互不打通、UI粗糙、无AI能力 |

### 2.2 核心机会缺口

经过分析，现有生态存在三个结构性缺口：

**缺口一：信息碎片化**
1.0 版本 27 页更新日志 + 50+ 新帕鲁 + 新区域 + 系统重构，信息分散在至少 5 个以上的内容平台。玩家需要跨平台拼凑才能获得完整认知。

**缺口二：人群无分层**
所有现有工具站对新玩家和 200 小时老玩家展示完全相同的内容。新手看到的是满屏 Tier List 和配种树，老玩家看到的是基础捕捉教程。信息与需求严重错配。

**缺口三：工具与内容割裂**
配种计算器是计算器，攻略是攻略，图鉴是图鉴，三者互不关联。玩家无法从"我卡在这个 Boss"直接跳到"我该用什么帕鲁配种来应对"，中间需要手动跨站检索。

### 2.3 我们的差异化定位

| 维度 | 传统Wiki/工具站 | 我们 |
|------|-----------------|------|
| 内容来源 | 人工编辑，更新慢 | AI 自动爬取全网 + 人工审核 |
| 展现形式 | 千人一面，静态列表 | 千人千面，按身份和意图动态分发 |
| 工具与内容 | 割裂，各管各 | 工具即内容，计算结果直接嵌入攻略 |
| 更新速度 | 1.0上线后数天至数周 | 1.0上线后数小时内完成数据入库 |
| 交互方式 | 查表式 | 对话式 + 查表式混合 |

## 三、产品设计与核心创新

### 3.1 "千人千面"个性化引擎

这是本项目最核心的创新点。系统根据三个维度动态组装内容：

**维度一：玩家身份标签**

| 标签 | 判定依据 | 内容偏好 |
|------|----------|----------|
| 纯新手 | 游戏时长<10h / 刚购买 | 入门路线、基础捕捉、早期帕鲁推荐、基地选址 |
| 进阶玩家 | 10-50h / 到达中期区域 | 配种入门、Boss攻略、基地自动化、坐骑选择 |
| 老手/回坑玩家 | 50h+ / 有EA存档 | 1.0变更影响分析、新区域探索、终局配种、禁猎区攻略 |
| 竞技/效率玩家 | 关注Tier List和数值 | 最优配种树、词条搭配、DPS计算、基地效率 |

身份标签的获取方式（Hackathon MVP 阶段）：
- 方式A：用户主动选择（"你是哪种玩家？"三按钮引导）
- 方式B：用户输入当前游戏进度（等级、已通关Boss、已有帕鲁），系统自动推断

**维度二：即时意图**

系统识别用户当前想解决的问题类型：

| 意图类型 | 典型表现 | 分发内容 |
|----------|----------|----------|
| 探索发现 | "世界树怎么去" | 区域攻略 + 前置条件 + 推荐帕鲁 |
| 战斗攻略 | "苍璃怎么打" | Boss机制 + 推荐配队 + 弱点分析 |
| 配种养成 | "怎么配出XXX" | 配种路径 + 词条选择 + 材料需求 |
| 基地建设 | "基地怎么布局" | 自动化方案 + 帕鲁分工 + 效率对比 |
| 版本变更 | "1.0改了什么" | 变更摘要 + 对老存档影响 + 适配建议 |

**维度三：游戏阶段感知**

根据用户输入的等级/进度，自动过滤不相关内容。例如等级 15 的玩家不会看到终局 Boss 攻略，而是看到"下一个该做什么"的引导。

### 3.2 核心功能模块（Hackathon MVP）

根据2天1夜的赛制，聚焦以下 5 个核心模块：

#### 模块一：AI 攻略聚合引擎

**功能定位：** 让【任何阶段的玩家】能【在3秒内获得与自身匹配的结构化攻略】。

- AI 爬虫自动抓取 B站、TapTap、17173、知乎等平台的帕鲁攻略内容
- LLM 清洗去重，提取结构化信息（帕鲁属性、配种路径、Boss机制、材料位置等）
- 按主题归类入库，标注内容质量分和时效性
- 前端按用户身份标签 + 意图动态组装展示

Hackathon 实现方式：
- 预置 20-30 篇高质量攻略作为种子数据（1.0 上线前用 EA 版本内容填充框架）
- 展示 AI 清洗后的结构化卡片（而非原文堆砌）
- 1.0 上线后接入实时爬虫管线（Demo 展示管线效果即可）

#### 模块二：智能帕鲁顾问（对话式）

**功能定位：** 让【任何玩家】能【用自然语言提问获得可执行的帕鲁养成建议】。

- 用户输入："我有 Lv30 疾风隼和 Lv25 烽歌龙，想配一个飞行坐骑"
- 系统返回：配种路径（疾风隼 x 烽歌龙 = XXX）、所需材料、预计时间、替代方案
- 与配种计算器数据打通，对话结果可直接查看详细配种树

这是对传统"选两个帕鲁看结果"式计算器的升级。用户不需要知道该配什么，只需要描述目标。

Hackathon 实现方式：
- 配种数据预置（EA 版本约 187 种帕鲁的配种关系表）
- LLM 做意图理解 + 数据检索 + 自然语言回答
- 1.0 新帕鲁数据上线后快速补录

#### 模块三：1.0 变更影响分析器

**功能定位：** 让【回坑老玩家】能【快速了解1.0对自己存档和帕鲁的具体影响】。

这是针对幻兽帕鲁 1.0 最具差异化的创新功能：

- 用户输入："我有这些帕鲁（贴名单/截图）"
- 系统分析：哪些帕鲁在 1.0 被调整（数值/技能/工作适应性变化）、哪些新帕鲁值得配种替代、新区域需要什么准备
- 输出：个性化"回坑指南"——你的存档在 1.0 该做什么

核心场景：
- "全球帕鲁箱"怎么用：哪些帕鲁该带到新存档
- 禁猎区改动对你已捕获帕鲁的影响
- 新增"人帕一体"机制哪些帕鲁可用
- 建造系统换色等 QoL 改动

Hackathon 实现方式：
- 预置 1.0 已公布的变更清单（27 页更新日志的关键提取）
- 用户勾选自己的帕鲁/进度
- 系统匹配变更项，输出个性化影响报告

#### 模块四：千人千面首页

**功能定位：** 让【首次访问的玩家】能【在5秒内找到与自己最相关的内容入口】。

- 三按钮引导："刚入坑 / 玩过EA版 / 硬核效率党"
- 根据选择动态渲染首页卡片（新手看入门路线，老手看1.0变更，效率党看Tier List）
- 每个卡片可展开查看详情，关联到对应工具

#### 模块五：核心数据工具集

**功能定位：** 让【需要查数据的玩家】能【在一个站内完成所有数据查询】。

集成以下工具（Hackathon 至少完成 2 个）：
- 配种计算器（双向：父母→后代 / 后代→父母）
- 帕鲁图鉴（属性、技能、工作适应性、掉落、刷新点）
- Boss 攻略卡片（机制、推荐配队、弱点）
- 坐骑/工作帕鲁 Tier List

## 四、技术架构概要

### 4.1 系统分层

```
┌─────────────────────────────────────────────┐
│              前端展示层（Next.js）             │
│   千人千面首页 / 对话式顾问 / 数据工具 / 攻略卡片  │
├─────────────────────────────────────────────┤
│            个性化分发引擎（API 层）             │
│   身份标签识别 / 意图理解 / 内容组装 / 阶段过滤   │
├─────────────────────────────────────────────┤
│              AI 内容管线（后端）               │
│   爬虫调度 → LLM清洗去重 → 结构化入库 → 质量评分  │
├─────────────────────────────────────────────┤
│              数据层（存储）                    │
│   帕鲁数据库 / 配种关系表 / 攻略知识库 / 变更日志  │
└─────────────────────────────────────────────┘
```

### 4.2 AI 内容管线（核心创新的技术支撑）

**Step 1：多源爬取**
- 数据源：B站视频简介/字幕、TapTap攻略帖、17173、知乎专栏、SMZDM、官方更新日志
- 方式：定时爬虫 + 关键词触发爬取
- Hackathon Demo：预置 20-30 篇抓取结果，展示管线效果

**Step 2：LLM 清洗结构化**
- 输入：非结构化攻略文本/视频字幕
- LLM 任务：提取帕鲁名、属性数据、配种关系、Boss机制、材料位置、推荐配队
- 去重：同一信息多源验证，冲突时以官方数据/高赞源为准
- 输出：结构化 JSON 入库

**Step 3：质量评分**
- 信息完整度：关键字段是否齐全
- 时效性：是否标注版本号，是否与 1.0 兼容
- 来源可信度：官方 > 高粉UP主 > 普通用户

**Step 4：个性化组装**
- 用户请求 → 身份标签 + 意图识别 → 从知识库检索匹配内容 → LLM 组装自然语言回答 + 结构化卡片

### 4.3 技术选型建议（Hackathon 适用）

| 层 | 技术 | 理由 |
|----|------|------|
| 前端 | Next.js + Tailwind | SSR 利于 SEO，组件生态成熟 |
| 后端 | Python FastAPI | 快速开发，AI 生态好 |
| LLM | GLM / 通义 / 内部AI平台 | 中文攻略理解能力强 |
| 数据库 | SQLite（Demo）/ PostgreSQL（正式） | Demo 轻量，正式可扩展 |
| 爬虫 | requests + BeautifulSoup / Playwright | B站等需动态渲染 |
| 向量检索 | 可选：FAISS / 内部 RAG 平台 | 攻略语义检索 |

## 五、幻兽帕鲁 1.0 专项差异化设计

针对幻兽帕鲁 1.0 正式版的独特内容，设计以下差异化功能（这些是其他游戏工具站没有的）：

### 5.1 全球帕鲁箱决策助手

1.0 新增"全球帕鲁箱"系统，允许跨存档转移帕鲁。这是玩家最纠结的决策点。

功能设计：
- 用户输入自己的帕鲁名单（或截图识别）
- 系统分析：哪些帕鲁值得带到新存档（基于 1.0 数值变化）、哪些可以留旧档
- 输出：推荐转移清单 + 理由（"这只帕鲁的词条在1.0仍然T0"/"这只被削弱了，不必带"）

### 5.2 1.0 新帕鲁速查雷达

1.0 新增 50+ 种帕鲁，包括莲花龙、电气狮子、拔刀鱼、Dupin 等已曝光角色。

功能设计：
- 新帕鲁卡片：属性、技能、工作适应性、获取方式（已知部分）
- 标注"待1.0上线后补录"的未确认信息
- 与配种计算器联动：新帕鲁能配出什么/怎么配出来

### 5.3 禁猎区 2.0 攻略

1.0 对禁猎区做了颠覆性改动：专属特殊环境、独有生态体系、能量屏障 + 巡逻防卫无人机。

功能设计：
- 各禁猎区改动对比（EA vs 1.0）
- 闯入策略：屏障破解方法、无人机规避路线
- 稀有帕鲁刷新点（1.0 新增）

### 5.4 人帕一体战斗配队

1.0 引入"人帕一体"创新机制——部分新帕鲁可化身为主角使用的武器。

功能设计：
- 人帕一体帕鲁列表 + 武器形态 + 战斗效果
- 配队推荐：人帕一体 + 传统战斗帕鲁的最优组合
- Boss 战适配：针对新塔主"苍璃"等 Boss 的配队方案

### 5.5 世界树/空岛云海探索路线规划

1.0 两大终局区域，玩家需骑乘巨型飞行帕鲁抵达。

功能设计：
- 区域地图 + 关键地标
- 前置条件清单（推荐帕鲁等级/坐骑类型/物资准备）
- 探索路线推荐（按完成度排序的检查清单）

## 七、商业模式与复制路径

### 7.1 商业模式验证（帕鲁打样阶段）

| 收入来源 | 说明 | 阶段 |
|----------|------|------|
| 广告/赞助 | 游戏外设、加速器、服务器租赁商 | MVP 阶段即可接入 |
 | 高级功能订阅 | AI 深度分析、无限对话、个性化配种路线规划 | 验证后 |
| 数据授权 | 向其他工具站/社区提供结构化数据 API | 长期 |
| 游戏联运 | 引导购买游戏本体（Steam Affiliate 等） | MVP 阶段 |

### 7.2 复制路径

帕鲁 1.0 验证"AI + 个性化"模式可行后，复制到其他热门游戏：

| 游戏 | 时机 | 理由 |
|------|------|------|
| 黑神话：悟空 DLC | DLC 发布时 | 单机+攻略需求强 |
| 明日方舟 / 原神 | 版本更新时 | 长线运营，数据维度丰富 |
| 三角洲行动 | 持续 | 用户已有 deltaforcetools.gg 基础 |
| 新热门游戏 | 发售窗口期 | 抢占内容空白期 |

复制核心：AI 内容管线 + 个性化引擎是通用基础设施，换游戏只需换数据源和领域知识。

## 八、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 1.0 上线时间与 Hackathon 时间冲突 | 无法用真实 1.0 数据 Demo | 用 EA 版本数据 + 1.0 已公布信息预置，Demo 展示管线能力 |
| AI 爬虫法律/版权风险 | 内容来源合规问题 | 仅抓取公开网页，标注来源，不存储原文，只存结构化数据 |
| LLM 回答准确性 | 给出错误配种建议 | 配种等硬数据走数据库精确匹配，LLM 只做意图理解和自然语言组装 |
| Hackathon 工期紧张 | 做不完 | 严格 P0/P1/P2 分级，P0 必须第一天完成 |
| 数据量不足 | Demo 效果差 | 运营提前预置 30+ 帕鲁完整数据和 20+ 篇攻略种子 |

### 9.2 游戏核心玩法循环

1. 开放世界探索（草原/雪山/火山/沙漠/禁猎区/世界树/空岛云海）
2. 帕鲁捕捉（潜行背袭提升成功率）
3. 帕鲁养成（配种牧场繁殖，词条遗传）
4. 自动化基地建设（帕鲁24小时流水线：采矿/伐木/种植/烹饪/发电）
5. 战斗挑战（野外Boss、高塔首领、禁猎区、世界树终局）
