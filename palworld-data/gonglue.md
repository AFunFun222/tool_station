# 幻兽帕鲁 6 个攻略总结
数据来源：`pals.json`、`pals_list.json`、`breeding_table.json`、`breeding_reverse.json`、`items.json`、`passive_skills.json`
## 1. 新手开荒必抓帕鲁攻略
前期开荒最重要的不是盲目抓稀有帕鲁，而是优先补齐基地自动化的基础岗位。从 `pals.json` 可以看到，像 Lamball 这类 1 星帕鲁虽然面板普通，但自带 `Handiwork`、`Transporting`、`Farming` 等基础工作适应性，属于最适合前期过渡的万用型劳动力。而 Cattiva 额外具备 `Mining`，Lifmunk 则覆盖 `Planting`、`Gathering`、`Lumbering`，这意味着新手只要尽快凑齐几只低稀有度、多工种帕鲁，就能明显提升木材、矿石、农场和搬运效率。
从数值上看，前期推荐优先抓取 `Rarity` 较低、`CaptureRateCorrect` 较高的帕鲁，这类目标更容易捕获，投入的帕鲁球成本也更低。开荒阶段不建议一开始就追求高战力终局宠，而是先围绕采集、搬运、种植、手工四条生产线补齐岗位。只要基地能稳定运转，后续装备、药剂和帕鲁球的产出都会顺畅很多。
## 2. 基地打工皇帝推荐攻略
如果你的目标是把基地效率拉满，那么判断一只帕鲁值不值得培养，核心要看两个维度：一是 `pals.json` 里的 `work_suitability`，二是 `stats['Work Speed']`。工作适应性决定它能干什么，工作速度决定它干得有多快。真正优秀的打工帕鲁，不一定是战斗最强的，但一定是在关键岗位上等级高、覆盖面广、吃饭少、掉 SAN 慢。
从数据结构来看，工作岗位主要包括 `Kindling`、`Watering`、`Planting`、`GeneratingElectricity`、`Handiwork`、`Gathering`、`Lumbering`、`Mining`、`Cooling`、`Transporting`、`Farming` 等。攻略思路上，建议玩家优先筛选单项等级高的专业工种帕鲁，再搭配少量多面手补空缺岗位。比如前期可以用低星多工种帕鲁撑起基础生产，中后期再逐步替换成高等级矿工、搬运工、点火工和流水线手工帕鲁。
另外，被动词条也会直接影响基地效率。`passive_skills.json` 中像 `Remarkable Craftsmanship` 提供 `Work Speed +75%`，`Lucky` 提供 `Work Speed +15%`，都属于极品打工词条；如果再叠加 `Nocturnal` 这类夜行能力，被培养出来的工作帕鲁几乎可以全天候运转。简单说，基地毕业阵容不是谁稀有带谁，而是岗位等级、工作速度、被动词条三项一起看。
## 3. 配种路线速成攻略
配种系统的核心价值，不是随机碰运气，而是通过固定公式稳定产出目标帕鲁。`breeding_table.json` 提供了完整的正向配种表，也就是亲本 A + 亲本 B = 子代；`breeding_reverse.json` 则提供反向查询能力，也就是想要某只子代，可以倒推出哪些父母组合能做出来。这两份数据结合起来，已经足够支撑一套完整的配种攻略。
实战中最推荐的思路不是从父母往下试，而是先确定你想要的目标帕鲁，再反查所有可行组合，然后根据自己现有库存选择成本最低的一条路线。进一步筛选时，还可以结合 `pals.json` 中的 `CombiRank` 和 `Egg` 字段：`CombiRank` 可以辅助判断配种稀有度和路线价值，`Egg` 则能帮助你提前判断产出的蛋类型，方便规划孵化资源。
如果你是新手，建议先把配种当成补功能位的工具，比如补飞行坐骑、补高等级工作帕鲁、补带优质被动的战斗宠；如果你已经进入中后期，配种的重点就会转向词条继承和终局毕业宠。配种真正强的地方，不是做出一只新帕鲁，而是把你想要的属性、功能和被动逐步定向组合出来。
## 4. 战斗强度榜与毕业宠培养攻略
判断一只帕鲁的战斗价值，不能只看单一攻击数值，而要综合 `pals.json` 中的 `Attack`、`Defense`、`Health`、`MeleeAttack`，以及 `level_65_stats` 的成长上限。因为很多帕鲁在低等级阶段差距不明显，但到了 65 级后，生命、攻击和防御的上下限会明显拉开，这也是为什么终局榜单通常更看重成长区间而不是初始面板。
除了基础属性，主动技能和被动技能同样决定实战上限。`active_skills` 中的 `power`、`cooltime`、`element` 可以直接反映技能爆发、循环和属性克制能力；而 `passive_skills.json` 里的 `Legend`、`Demon God`、`Diamond Body`、各类元素增伤词条，则决定了一只帕鲁是否具备真正的毕业潜力。比如 `Legend` 同时提升攻击、防御和移动速度，属于泛用毕业词条；`Demon God` 更偏极限输出；元素增伤词条则适合围绕属性队构筑。
因此，战斗攻略的正确思路不是抓到高稀有就毕业，而是先筛出成长面板优秀的帕鲁，再通过配种和词条继承，把高攻击、高生存和高技能质量叠加到同一只个体上。真正的 T0 战斗宠，往往是高成长面板、强主动技能、顶级被动词条三者同时成立。
## 5. 资源掉落与材料 farming 攻略
很多玩家中后期卡进度，不是因为打不过，而是因为材料链断了。`pals.json` 中每只帕鲁都带有 `drops` 字段，记录了可掉落物、数量和概率；`items.json` 则进一步给出了大量物品的 `categories`、`crafting_recipes` 和 `drop_sources`。这意味着你完全可以从我要做什么装备反推我该去刷什么材料，以及这些材料由谁掉落。
攻略上最实用的方法，是先锁定目标装备或消耗品，再拆解其制作配方中的关键材料。例如某些高级装备需要大量矿物、器官、皮革或特殊掉落物，那么就可以根据 `drop_sources` 和帕鲁掉落表，整理出一条高效率 farming 路线。这样做的好处是，你不会再陷入缺什么刷什么、刷完又缺别的的低效循环，而是能围绕一整条生产链集中刷取。
如果把基地自动化和掉落 farming 结合起来，效率会更高：基础材料交给基地生产，稀有材料集中去刷特定帕鲁或 Boss。对大多数玩家来说，真正提升资源效率的关键不是单次掉率，而是目标明确、路线固定、生产链闭环。
## 6. 回坑玩家版本追赶攻略
回坑玩家最容易犯的错误，是沿用旧版本思路直接继续养旧阵容，而忽略了新帕鲁、新配种路线和新终局内容带来的环境变化。结合 `pals.json` 与 `breeding_reverse.json` 可以发现，很多帕鲁已经不只是原版形态，还存在大量带后缀的变种或区域版本，例如名称中常见的 `Noct`、`Cryst`、`Ignis`、`Terra`、`Lux`、`Aqua` 等。这类变种通常意味着属性、定位或适用场景发生了变化，回坑时必须重新认识。
另一方面，配种系统的价值在版本推进后会越来越高。以前难以获得的功能型或终局型帕鲁，现在可能已经能通过更多父母组合稳定做出来。再结合 `passive_skills.json` 中的高价值词条，例如 `Legend`、`Swift`、`Artisan`、`Vampiric` 等，回坑玩家完全可以跳过一部分旧时代的重复培养流程，直接围绕新环境重建阵容。
所以回坑的正确顺序应该是：先确认当前版本值得追的功能位和战斗位，再检查自己旧档里哪些帕鲁还能继续用，哪些已经被替代，最后利用新的配种路线和词条体系快速补齐短板。这样做比盲目重开或死守旧阵容都更高效。
